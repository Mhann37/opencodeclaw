## Health Briefing System (FIXED)

**Issue resolved:** Heath agent cron (07:30 AEDT) was failing due to WHOOP API returning malformed JSON with dotenv logging contaminating stdout.

**Root cause:** 
- WHOOP CLI mixes stdout + stderr (dotenv messages prepend JSON)
- WHOOP scores sleep overnight, so 07:30 call may fetch today's data (not yet scored, returns empty array)
- Script was trying to parse corrupted JSON without proper error handling

**Solution implemented (v3):**
- Modified `heath-briefing-runner.py` to:
  1. Fetch YESTERDAY's data (which is scored by morning)
  2. Combined stdout + stderr parsing to extract JSON cleanly
  3. Retry logic: 3 attempts with 5s delay between
  4. Cache fallback if API fails (stores valid WHOOP data for future retries)
  5. Graceful degradation: Shows "WHOOP unavailable" vs silent N/A values
- Test run (2026-03-22 07:33 AEDT) succeeded on 2nd retry with valid metrics:
  - Recovery: 68%
  - HRV: 44.2ms
  - RHR: 56bpm

**Status:** Production-ready. Tomorrow's 07:30 run should deliver clean full briefing on first attempt (assuming normal wake time, so WHOOP data is scored).

## WineNight App — Commercial Strategy (MAJOR DECISION)

**Pivot approved:** Shift from subscription to hybrid pricing + winery B2B focus.

### New Monetization Model

**Personal Tier (D2C):**
- **Free:** Up to 4 participants per tasting (with lightweight ads: wine retailers, clubs)
- **One-time purchase:** $9.99 for unlimited participants + AI breakdown + export (no ads, lifetime access)
- **Rationale:** One-time payment >> subscription friction; 10-15% conversion expected (vs. 3-5% for subscription)

**Winery Tier (B2B) — THE REAL BUSINESS:**
- **Free trial:** 1 tasting/month (limited analytics)
- **Pro:** $99/month (unlimited tastings, staff dashboard, customer email capture, real-time analytics, branded results)
- **Enterprise:** Custom pricing, white-label, CRM integrations, multi-location
- **Rationale:** Wineries have clear ROI (3x payback: $99/mo cost → $240+/mo extra revenue from upsells + email follow-up)

### Cellar Door Value Prop

**Current winery pain points:**
- Manual scorecards (paper, lost data)
- No customer data capture (30% email collection vs. 90% with app)
- No follow-up (generic emails, zero personalization)
- Staff time wasted on manual tallying + data entry
- Zero insights into customer preferences

**WineNight solves:**
1. Staff creates session on iPad → prints QR code at table
2. Customers scan QR, score wines in app
3. Staff sees real-time results + which wines are winning
4. Emails captured automatically (no signup form friction)
5. Post-tasting: Automated personalized email ("Your profile: prefers full-bodied reds. Try [wine]. Buy here →")
6. Follow-up sales: 12-15% of tasters make post-visit purchases ($25-30 avg)
7. Wine club signups: 3-5 per 75 tasters per month

**ROI for typical boutique winery (50-100 tasters/month):**
- Extra revenue from follow-up emails: $250/month
- Wine club signups: $90/month recurring
- **Total: $340/month extra revenue vs. $99/month cost = +$241/month net** (3x ROI)

### Go-to-Market Strategy (REVISED)

**Original plan:** Personal tier first (crowded, low LTV, hard to acquire)

**New plan:** Wineries first → Personal second
1. **Weeks 1-2:** Adapt app for winery use (add auth, staff dashboard, QR codes, real-time analytics)
2. **Weeks 3-4:** Winery beta with 5-10 local pilot customers (free trial)
3. **Weeks 5-8:** Cold outreach to 50+ regional wineries (NSW, VIC, WA), target 5-10 paying customers
4. **Weeks 9-12:** Launch personal tier (use winery success as social proof)

**Why this order:**
- Wineries fund development (recurring $99/mo revenue)
- Personal is "bonus" (one-time $9.99, high margin, low CAC)
- Wineries = market validation + case studies for personal marketing
