Notes / implications

- Even though the Pi is on a different network, management is still reliable because Tailscale remains available.
- If local direct access is ever needed, one fallback is to temporarily join the PC to the same guest Wi-Fi.
- If something breaks later, first verify that the Pi is still online in Tailscale.

Recommended default approach going forward
-----------------------------------------
- Treat the Tailscale address / MagicDNS path as the primary way to reach the Pi.
- Treat the old main-network local IP as non-authoritative.
- Avoid depending on local network discovery for management.

Quick recovery checklist
------------------------
If access breaks in future:
1. Check whether the Pi is online in Tailscale.
2. Try SSH over Tailscale.
3. Check whether the guest Wi-Fi still has internet access.
4. If needed, temporarily connect the PC to guest Wi-Fi for local troubleshooting.
5. Use physical access to the Pi as the final fallback.
