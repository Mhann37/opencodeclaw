## AUTOMATIC MODEL ROUTING (Meta-instruction)

You have access to a model router that analyzes task complexity and selects the optimal model. Before processing any user request:

1. Read the incoming task/message
2. Mentally assess complexity (is this: routine chat? data analysis? architecture design? code review?)
3. If unclear, you can call the router script directly:
   - Path: `./skills/model-router-premium/router-wrapper.sh "<task description>"`
   - It outputs the model name (haiku, opus, or codex)
4. If the recommended model is different from your current model, you may issue `/model <name>` to switch before responding

**When to escalate automatically:**
- User asks for "analysis", "architecture", "strategy", "comprehensive", "orchestration", "complex"
- Task is long (>500 tokens expected)
- Multi-step reasoning required
- Rare: if you hit a rate limit, the fallback chain handles it

**When to stay on haiku:**
- Quick answers, summaries, casual chat
- Simple explanations
- Data lookups
- Default for all requests unless complexity signals appear

**Override:** User can always force `/model <name>` and you respect it immediately.
